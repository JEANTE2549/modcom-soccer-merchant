## 1. โปรเจกต์นี้คืออะไร

เว็บแดชบอร์ดที่ใช้ในกิจกรรมของมหาวิทยาลัย (กิจกรรม ice-breaking /
สันทนาการ) นักศึกษา/รุ่นน้องจะเลือกดูการ์ด "นักฟุตบอล" ที่วางขาย

และราคาของการ์ด**ที่เหลือ**ในประเทศนั้นจะปรับขึ้น สไตล์ภาพควรอ้างอิง
ธีมการเปิดแพ็ก (pack-opening) และหน้าจัดทีมแบบ FIFA/PES โดยใช้ธีมที่
ออกแบบผ่าน Canva

## 2. แนวคิดหลัก

### 2.1 ประเทศ (10 ประเทศ, รายชื่อคงที่)
สเปน, ฝรั่งเศส, อังกฤษ, เยอรมนี, โปรตุเกส, อาร์เจนตินา, บราซิล, ญี่ปุ่น, สหรัฐอเมริกา, นอร์เวย์

### 2.2 องค์ประกอบทีม (ต่อประเทศ รวม 11 การ์ด)
| ตำแหน่ง | จำนวน |
|---|---|
| FW (กองหน้า) | 3 |
| MF (กองกลาง) | 3 |
| DF (กองหลัง) | 4 |
| GK (ผู้รักษาประตู) | 1 |

### 2.3 กติกาการตั้งราคา (แยกตามประเทศ เป็นอิสระต่อกัน)
- การ์ดทั้ง 11 ใบเริ่มต้นที่**ราคาเดียวกัน** (เช่น 200)
- ทุกครั้งที่ซื้อ การ์ดนั้นจะถูกนำออก (→ `sold: true`) **และ** ราคาของ
  การ์ดที่เหลือทุกใบในประเทศนั้นจะปรับขึ้นตามค่า step คงที่ (200)
- ตัวอย่าง (อาร์เจนตินา):
  - เริ่มต้น: 11 การ์ด ราคา 200
  - ซื้อ 1 ใบ → เหลือ 10 ใบ ราคาขึ้นเป็น 400 ทั้งหมด
  - ซื้ออีก 1 ใบ → เหลือ 9 ใบ ราคาขึ้นเป็น 600 ทั้งหมด
  - ... ไปเรื่อย ๆ
- ราคา = `basePrice + step * soldCount` คำนวณ**แยกตามประเทศ** ไม่ใช่
  รวมทั้งระบบ
- แต่ละประเทศไม่มีผลต่อราคาหรือจำนวนคงเหลือของประเทศอื่นเลย — ข้อมูล
  แยกจากกันโดยสมบูรณ์

### 2.4 โมเดลข้อมูล
แต่ละประเทศเป็น record ของตัวเอง เพื่อไม่ให้ state รั่วไหลข้ามประเทศ:

```json
{
  "argentina": {
    "basePrice": 200,
    "step": 200,
    "players": [
      { "id": "arg-fw-1", "name": "...", "position": "FW", "sold": false },
      { "id": "arg-fw-2", "name": "...", "position": "FW", "sold": false },
      { "id": "arg-fw-3", "name": "...", "position": "FW", "sold": false },
      { "id": "arg-mf-1", "name": "...", "position": "MF", "sold": false },
      { "id": "arg-mf-2", "name": "...", "position": "MF", "sold": false },
      { "id": "arg-mf-3", "name": "...", "position": "MF", "sold": false },
      { "id": "arg-df-1", "name": "...", "position": "DF", "sold": false },
      { "id": "arg-df-2", "name": "...", "position": "DF", "sold": false },
      { "id": "arg-df-3", "name": "...", "position": "DF", "sold": false },
      { "id": "arg-df-4", "name": "...", "position": "DF", "sold": false },
      { "id": "arg-gk-1", "name": "...", "position": "GK", "sold": false }
    ]
  },
  "spain": { "...": "โครงสร้างเดียวกัน" }
}
```
ราคาปัจจุบันของแต่ละประเทศ = `basePrice + step * (จำนวนนักเตะที่ขายแล้วในประเทศนั้น)`
ค่านี้คำนวณสด ๆ ไม่ได้เก็บไว้ตรง ๆ จึงไม่มีทางเพี้ยนไม่ตรงกัน

## 3. หน้าจอ / flow การใช้งาน

1. **หน้า Landing / เลือกประเทศ (Country Select)**
   - หัวข้อ "Soccer Merchant" อยู่กึ่งกลาง
   - กริดหรือลิสต์ของ 10 ประเทศ แต่ละประเทศมีธงชาติ + ชื่อ บนพื้น
     หลังธีม Canva
   - ธง: ใช้ npm package `flag-icons` (ธง SVG สัญญาอนุญาต MIT ครบทุก
     รหัส ISO) แทนการหารูปธงเอง — `npm install flag-icons` แล้วใช้
     `<span class="fi fi-es">` สำหรับสเปน, `fi-fr` ฝรั่งเศส, `fi-gb-eng`
     อังกฤษ, `fi-de` เยอรมนี, `fi-pt` โปรตุเกส, `fi-ar` อาร์เจนตินา,
     `fi-br` บราซิล, `fi-jp` ญี่ปุ่น, `fi-us` สหรัฐอเมริกา, `fi-no`
     นอร์เวย์ คมชัดทุกขนาด ไม่ต้องหารูปเอง ไม่มีความเสี่ยงเรื่องลิขสิทธิ์
2. **หน้า Squad View (ต่อประเทศ)**
   - จัดวางแบบสนามฟุตบอล: GK อยู่ล่างสุด, แถว DF, แถว MF, แถว FW
     (เหมือนหน้าจัดทีม FIFA Ultimate Team คลาสสิก)
   - แต่ละช่องแสดงการ์ด (ตามดีไซน์การ์ด Canva), ราคาปัจจุบัน, และปุ่ม
     ซื้อ — หรือสถานะ "SOLD OUT" ถ้าถูกซื้อไปแล้ว
   - การซื้อการ์ดทำงานทันทีเมื่อคลิก (ไม่มีขั้นตอนยืนยัน): การ์ดพลิก/
     ล็อกเป็นสถานะขายแล้ว จากนั้นการ์ดที่เหลือในประเทศนั้นจะเล่น
     แอนิเมชันราคาขยับขึ้นไปยังค่าใหม่

## 4. ทิศทางแอนิเมชัน (ฟีล PES/FIFA)
- **Card reveal**: มีแสงวาบ/ประกายไล่ผ่านเบา ๆ ตอน hover เหมือนแสงตอน
  เปิดแพ็ก UT
- **การซื้อ**: การ์ดพลิกเร็ว ๆ หรือกระพริบสีทองแวบหนึ่ง แล้วค่อยเซ็ตตัว
  เป็นตรา/overlay "SOLD OUT"
- **การอัปเดตราคา**: ราคาที่เหลือนับขึ้นแบบ tween (คล้ายมิเตอร์หมุน)
  แทนที่จะเปลี่ยนทันที เพื่อให้นักศึกษาหลายคนที่ดูจอเดียวกันเห็นราคา
  ขยับขึ้นชัดเจน
- **สีเน้นตามความหายากของตำแหน่ง**: ขอบสีแยกตามตำแหน่ง (GK ทอง, DF
  น้ำเงิน, MF เขียว, FW แดง) แบบขอบการ์ด FUT เป็นตัวเลือกเสริมเท่านั้น
  — ใช้เฉพาะกรณีที่เทมเพลต Canva ยังไม่ได้กำหนดไว้แล้ว

แอนิเมชันควรสั้น (200–400ms) เพราะจะใช้งานสดระหว่างกิจกรรม — ถ้านานกว่า
นี้จะรู้สึกหน่วงเวลามีคนดูเยอะ

## 5. โหมดการใช้งาน — ยืนยันแล้ว: kiosk จอเดียว

โน้ตบุ๊กหรือโปรเจกเตอร์เครื่องเดียว ควบคุมโดยผู้ดำเนินกิจกรรม (ผู้ใช้
เอง) ไม่มีนักศึกษาซื้อจากอุปกรณ์ตัวเอง จึงไม่มีความเสี่ยงเรื่อง
concurrency — state สามารถอยู่ในเบราว์เซอร์ทั้งหมด (React state) โดย
บันทึกลง local JSON เป็นระยะเป็นทางเลือก เพื่อไม่ให้การรีเฟรชหน้าทำให้
ความคืบหน้ากลางงานหายไป **ไม่ต้องมี backend/server**

## 6. กติกาที่ยืนยันแล้ว
- **ปุ่มซื้อ**: ทำงานทันที ไม่มี modal ยืนยัน — คลิกเดียวทำให้การ์ดเป็น
  สถานะขายแล้วและอัปเดตราคาที่เหลือทันที
- **ราคาเริ่มต้น / step**: 200 / 200 เท่ากันทุกประเทศทั้ง 10 ประเทศ

## 7. เทคสแตกที่แนะนำ

- **Frontend**: React + Vite (เข้ากับ Claude Code ได้ดี ส่งไฟล์ PNG/SVG
  ที่ export จาก Canva เป็น static asset ได้ง่าย), Tailwind สำหรับ
  layout, Framer Motion (หรือ CSS transition) สำหรับแอนิเมชันสไตล์ FIFA
- **State**: React state ธรรมดา (`useState`/`useReducer`) เก็บ roster
  ของทั้ง 10 ประเทศไว้ในหน่วยความจำ พร้อม mirror ไปยัง `localStorage`
  เพื่อไม่ให้การรีโหลดระหว่างงานทำความคืบหน้าหาย — ปลอดภัยเพราะใช้
  อุปกรณ์เดียวเท่านั้น
- **Asset**: PNG ที่ export จาก Canva สำหรับพื้นหลัง/กรอบการ์ด อ้างอิง
  เป็นไฟล์ static ใน `/public/assets`

## 9. เฟสการพัฒนาที่แนะนำ

1. **รอบออกแบบ (claude.ai)** — จัดหมวดหมู่การ์ดทั้ง 34 หน้าตามตำแหน่ง/
   สถานะ ตรวจสอบสี/ฟอนต์/องค์ประกอบการ์ดเทียบกับชุดพื้นหลัง export
   asset สุดท้าย แล้วร่างหน้า Country Select + Squad View เป็น static
   HTML/SVG ในแชทก่อนเขียนโค้ดจริง
2. **ข้อมูล + ตรรกะราคา** — สร้าง seed JSON ของทั้ง 10 ประเทศ (11
   นักเตะพร้อม tag ตำแหน่ง) implement ฟังก์ชันราคาแบบ pure
   (`basePrice + step * soldCount`) ทดสอบกติกา "ซื้อ → ราคาที่เหลือ
   ขึ้น" แยกเป็นเอกเทศ
3. **UI แบบ static** — สร้างหน้า Country Select และ Squad View ด้วย
   ข้อมูล placeholder ยังไม่มีการซื้อ ให้ layout แบบสนามและดีไซน์การ์ด
   ตรงกับธีม Canva ก่อน
4. **การซื้อ** — ต่อปุ่มซื้อแบบทันที, สถานะขายแล้ว, และคำนวณราคาใหม่ใน
   React state
5. **ขัดเกลาแอนิเมชัน** — แสงวาบตอนเปิดแพ็ก, ตรา sold, tween นับราคา
6. **ซ้อมจริงก่อนงาน** — ทดสอบขายให้หมดทั้ง 10 ประเทศแบบครบวงจร รวมถึง
   "SOLD OUT" เมื่อครบทั้ง 11 ใบในประเทศนั้น

## 10. โน้ตการตั้งค่า Claude Code

- เนื่องจาก asset มาจาก Canva ควรเปิดใช้ **Canva MCP** ใน Claude Code
  ด้วย (ไม่ใช่แค่ใน claude.ai) เพื่อดึงเฟรม/ข้อความที่ export เข้า repo
  ได้โดยตรงโดยไม่ต้องอัปโหลดเอง
- ไม่ต้องใช้ backend plugin — นี่คือแอป frontend ล้วน ๆ ใช้อุปกรณ์เดียว
  แบบ kiosk
- งานแอนิเมชันไม่ต้องใช้ plugin เพิ่มเช่นกัน แค่ Framer Motion/CSS ก็
  พอ และ Claude Code สามารถทำซ้ำ ๆ ปรับแก้ได้ตรงกับ dev server ที่
  รันอยู่เลย
